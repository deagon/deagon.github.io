openstack-thread 模型

Tech
  • openstack
  • thread

note.refers.link

Threading model

  • 所有 openstack 服务使用 green thread 模型，通过 eventlet 和 greenlet 库实现；
  • green thread 使用 cooperative （协程）模型；
  • 当且仅当某些 eventlet 或者 greenlet 被调用（比如 sleep, IO 调用等）时，线程会被切换；
  • 从操作系统的角度来看，每个 openstack 服务在一个单一线程下运行；
  • green thread 的使用减少了竞争条件的可能性，但没有完全消除；
  • 有些时候，你需要使用 @lockutils.synchronized(…) decorator 去避免竞争；

Yielding the thread in long-running tasks

  • 如果一段代码执行了太长时间而且没有包含任何会触发 eventlet 内容切换的方法，这个线程将会阻塞所有等待的线程；
  • 这个场景可以通过在长时间运行的代码中增加 eventlet sleep 调用的方式去避免；
  • 这个 sleep 可以触发一个线程切换操作，而且参数 0 不会引入额外的时间花费；
  • from eventlet import greenthread
  • greenthread.sleep(0)

Mysql access and eventlet

  • openstack 服务使用的是扩展 C library 访问 mysql, 所以对于 mysql 的查询将会阻塞服务的主线程；
  • 由于 eventlet 不能使用 monkey-patching 去中断在 C library 中的 blocking call，使得 mysql 查询会阻塞这个线程；
  • 在 Diablo 的 release 版中包含了一个不会引起阻塞的线程池的实现，但是这个实现引起了一个 bug，所以就被删除了；
